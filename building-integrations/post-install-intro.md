# Building integrations

## What just opened up
Every integration you connect from now on gets the full treatment automatically. Runneth reads who's setting it up and what the immediate job is, chooses the right depth (quick answer vs. full deep dive), pulls community intelligence, runs live verification against the connected account, applies the protocol matching the integration type, and solves any quirks it finds before you ever see them.

## Try this now
1. **Connect an integration and watch the flow**: `Connect Notion.`
   _You'll get back:_ the OAuth handoff, then a layered response: immediate capability overview, type-aware deep dive (workspace/org integrations get context sweep + write-back target discovery), and a type-specific activation proposal.
2. **Ask it to deep-dive an already-connected integration**: `Run the full deep dive on my Meta integration.`
   _You'll get back:_ community sweep across CLIs, MCP servers, and issue threads; live verification against your account; type-specific protocol (performance data gets schema design + compound analysis commands); practical guide written.
3. **See what type a new integration would fall under**: `If I connect Klaviyo, what type is it and what would the activation look like?`
   _You'll get back:_ classification (performance / attribution / capability / workspace / customer-BI), the matching protocol, and the activations Runneth would propose.

## Compounds with
- **integration-capabilities-library:** Every integration this builds gets a capabilities-and-scopes file the library keeps fresh nightly.
