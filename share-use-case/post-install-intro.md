# Share use case

## What just opened up
You can now package any working pattern from this sandbox into a use case other teams can install. Runneth walks you through naming the human outcome, mapping the install inventory, surfacing tokens, generating the files, validating across product, design, and engineering lenses, and producing a zip ready to submit to the library.

## Try this now
1. **Start from scratch on a new idea**: `I want to build and export a use case.`
   _You'll get back:_ a first question about what human outcome the use case enables, then a guided flow through inventory, tokens, file generation, validation, and a final zip.
2. **Improve an existing staged use case**: `Export my <use-case-id> use case.`
   _You'll get back:_ if it's already staged, Runneth identifies what's different from the last version, bumps the right semver, and walks the update path. If it's new, it starts the build flow.
3. **Get the design and PM review without exporting**: `Run the staff designer review on my <use-case-id>.`
   _You'll get back:_ a consolidated findings table from the design, PM, and engineering lenses with priority labels (fix now, next version, acceptable risk).

## Compounds with
- **plan-mode:** Building a use case is itself a build, so plan mode kicks in for the structural decisions before any file gets written.
